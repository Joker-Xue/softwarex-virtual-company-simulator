# 虚拟公司职业探索系统实践流程说明

本系统面向在校学生、职业规划课程参与者以及需要进行岗位认知训练的初学者，提供一个可交互、可观察、可解释的虚拟公司环境，用于将抽象的职业描述转化为具体的工作过程体验。用户首先创建或选择一个虚拟角色档案，设置昵称、目标部门、MBTI 类型以及沟通力、领导力、创造力、技术能力、团队协作和勤勉度等六项属性。系统随后根据角色配置进入虚拟公司场景，并在三层办公空间中呈现不同职能区域：一层包含大厅、咖啡区和人力资源部门，二层包含工程、产品、市场、财务和运营等业务部门，三层包含会议室、主管办公室和 CEO 办公室。公司中预置了覆盖不同 MBTI 类型、部门、职业等级和管理/技术成长路径的 NPC 智能体，例如工程、产品、市场、财务、人力、运营和管理层角色，使学生能够在同一组织环境中比较不同人格与岗位组合下的工作行为差异。

在模拟过程中，用户可以以第一人称职业探索者的身份进入公司，观察不同 NPC 在日常任务、会议、休息、社交互动和动态公司事件中的行为变化。系统的后台仿真循环会周期性驱动智能体行动：NPC 会根据 MBTI 偏好、当前任务、日程安排、部门位置、职业等级和事件吸引力，在工作、聊天、休息、会议、移动等行为之间进行选择。例如，内倾型和判断型角色更容易表现出较高的工作专注度和任务完成效率，外倾型或情感型角色在社交互动和关系增长上更活跃；直觉型角色在复杂或创新任务中更容易获得经验加成，思考型角色更倾向于技术路径，情感型角色则更倾向于管理路径。不同部门还会生成差异化任务，如工程部门的接口重构、单元测试和性能优化，产品部门的需求评审和 PRD 撰写，市场部门的活动策划和用户数据分析，财务部门的预算审批和风险报告，人力部门的简历筛选、面试和入职计划等。用户因此不仅能看到“某个岗位是什么”，还可以观察该岗位在虚拟工作日中的任务结构、互动对象、工作地点和成长方式。

系统输出不仅包括可视化的虚拟公司运行画面，还包括多层次的可解释诊断信息。用户可以在角色面板中查看个人职业等级、经验值、任务完成情况、好友关系、聊天记录、公司活动和数据分析结果；在诊断界面中，系统进一步展示人格参数如何影响行为决策，包括工作速度倍率、社交收益倍率、经验获取倍率、职业路径倾向、行为概率分布、偏好的任务标签、事件参与倾向以及最近的决策时间线。每次任务选择、任务完成、社交互动和事件参与都会被记录为行为 trace，包含偏好分数、匹配原因、经验收益、社交关系变化和位置变化等信息。系统还提供个人与公司两个层面的数据看板，例如个人经验增长趋势、任务状态分布、好友亲密度、部门人数分布、平均经验值和职业等级分布，从而帮助用户从“过程观察”和“数据解释”两个角度理解职业体验。

在实践使用上，该系统可作为职业规划课程中的课堂演示工具，也可用于小规模内部试用或职业探索工作坊。教师可以让学生分别选择不同 MBTI 类型和部门角色，进入同一个虚拟公司观察若干个虚拟工作日，再比较不同角色在任务偏好、社交频率、会议参与、成长路径和岗位日常行为上的差异。学生也可以通过重复创建不同角色或观察不同 NPC，比较“工程师”“产品经理”“市场人员”“财务人员”“HR”“运营人员”“管理者”等岗位在工作内容和组织互动中的具体差别。与静态职业介绍相比，该系统提供了一个动态、可 inspect 的职业体验过程：学生不仅阅读岗位描述，还能看到角色如何行动、为什么这样行动、哪些人格参数影响了决策，以及这些行为如何逐步积累为任务偏好、社交关系和职业路径变化。这使系统能够支持课堂讨论、职业兴趣澄清和岗位认知训练，帮助用户更具体地理解不同岗位的日常工作差异和自身特质与岗位环境之间的匹配关系。
